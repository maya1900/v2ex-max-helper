# 常见问题（FAQ）

## Q1. 提示「无 Cookie」或签到显示「Cookie 已失效」

- 确认已执行过 `node v2ex-checkin.js --save-cookie` 并生成了 `~/.v2ex_cookie`。
- Cookie 会过期，需重新登录 V2EX 复制最新 Cookie 再保存一次。
- 建议开启 `--ping` 保活心跳，减少过期频率。

## Q2. 阅读脚本报「Cookie 文件不存在或为空」

阅读模块与签到模块共用 `~/.v2ex_cookie`。先用签到模块的 `--save-cookie` 保存 Cookie。

## Q3. 无头服务器运行阅读脚本报错 / 浏览器起不来

`browser.js` 默认有头模式（`headless: false`）以降低风控。无头 VPS 请用 `xvfb`：

```bash
sudo apt-get install -y xvfb
xvfb-run -a node main.js
```

或将 `reader/browser.js` 中 `headless: false` 改为 `true`（风控概率略升）。

## Q4. 遇到 Cloudflare 「Just a moment...」挑战

脚本内置检测与最多 60 秒等待（`waitForCloudflare`）。若频繁卡住：
- 降低阅读频率；
- 更换干净 IP 的 VPS；
- 确保使用有头模式 + 真实 UA。

## Q5. Telegram Bot 没有反应

- 检查 `~/.v2ex_env` 中 `TG_TOKEN` 和 `TG_CHAT_ID` 是否正确。
- Bot 只响应 `TG_CHAT_ID` 对应的账号，用别的账号发消息会被忽略。
- 确认 `bot.js` 进程在运行（`systemctl status v2ex-bot`）。

## Q6. `/debug` 提示「日志文件不存在」

`bot.js` 默认读 `/var/log/v2ex-reader.log`。请确保阅读脚本的输出重定向到该路径，
或在 `~/.v2ex_env` 中设置 `READER_LOG=` 指向你的实际日志文件。

## Q7. 余额一直不变 / 活跃度奖励没触发

- V2EX 活跃度奖励有自身规则与上限，并非读得多就一定加币。
- `MAX_CHANGE_COUNT` 默认为 2，达到即认为奖励已发完并停止，属正常行为。
- 用 `node inspect_balance.js` 可打印 `/balance` 页面片段排查解析问题。

## Q8. 会不会泄露我的隐私？

- 代码中不含任何 Token、Chat ID、Cookie，全部来自本地配置文件。
- `.gitignore` 已排除 `.v2ex_env`、`.v2ex_cookie`、`data/`、`chrome-profile/` 等敏感内容。
- 切勿把 `~/.v2ex_cookie`、`~/.v2ex_env` 或 `reader/data/` 上传到任何公开仓库。
