# 配置说明

所有敏感配置均通过 **环境变量** 或 **`~/.v2ex_env` 文件** 提供，代码中不含任何个人信息。
环境变量优先级高于 `~/.v2ex_env` 文件。

## 配置项一览

| 变量名 | 用于 | 必填 | 说明 |
|--------|------|------|------|
| `TG_TOKEN` | reader（bot/notify） | Bot 必填 | Telegram Bot Token |
| `TG_CHAT_ID` | reader（bot/notify） | Bot 必填 | 唯一授权 Chat ID，Bot 硬锁只响应它 |
| `TG_BOT_TOKEN` | checkin | 可选 | 签到脚本推送用的 Bot Token |
| `BARK_URL` | checkin | 可选 | Bark 推送地址，如 `https://api.day.app/你的KEY` |
| `READER_LOG` | reader（bot） | 可选 | 阅读日志路径，`/debug` 命令读取，默认 `/var/log/v2ex-reader.log` |
| `COOKIE_FILE` | 全部 | 可选 | Cookie 文件路径，默认 `~/.v2ex_cookie` |
| `V2EX_COOKIE` | checkin（--save-cookie） | — | 仅在保存 Cookie 时临时传入 |
| `DB_PATH` | reader（queue） | 可选 | SQLite 数据库路径，默认 `reader/data/queue.db` |

## `~/.v2ex_env` 示例

```ini
TG_TOKEN=<在此填入你的BotToken>
TG_CHAT_ID=<在此填入你的Chat_ID>
TG_BOT_TOKEN=<在此填入你的BotToken>
# BARK_URL=https://api.day.app/你的KEY
# READER_LOG=/var/log/v2ex-reader.log
```

> 文件格式为 `键=值`，每行一条，键必须是大写字母和下划线。务必 `chmod 600 ~/.v2ex_env`。

## 阅读脚本可调参数

在 `reader/main.js` 顶部：

| 常量 | 默认 | 含义 |
|------|------|------|
| `MAX_READ_COUNT` | 1000 | 每日阅读篇数上限（安全兜底） |
| `MAX_CHANGE_COUNT` | 2 | 余额变化达到此次数即停止（活跃度奖励通常两轮） |
| `BALANCE_CHECK_INTERVAL` | 50 | 每读多少篇检查一次余额 |
| `DEADLINE_UTC_HOUR` | 6 | 截止时间（UTC 06:00 = 北京 14:00），超时强制退出 |

阅读停留时间在 `reader/browser.js` 的 `readPost()` 中（默认 12~18 秒随机）。
